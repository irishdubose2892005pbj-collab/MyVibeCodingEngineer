# Stage-1: Vibe Coding

> 用 AI 作为搭档，让编程像说话一样自然

**预计耗时**: 2-4 周  
**产出目标**: 3+ 个可运行的 AI 辅助项目原型

---

## 🎯 阶段目标

1. 理解 Vibe Coding 的核心范式：自然语言驱动开发
2. 掌握与 AI 结对编程的高效协作模式
3. 学会用 Dify 等低代码平台快速搭建 AI 应用
4. 建立"Prompt → 代码 → 测试 → 迭代"的快速闭环

---

## 📋 学习任务清单

### 01. 贪吃蛇游戏
**目标**: 用 AI 在 1 小时内写出一个可玩的贪吃蛇

- [ ] 向 AI 描述需求，生成基础版本
- [ ] 迭代优化：添加分数、关卡、音效
- [ ] 学习如何调试 AI 生成的代码
- [ ] **产出**: 一个可在线访问的贪吃蛇游戏

### 02. AI 能力边界
**目标**: 了解 AI 编程的擅长点和局限

- [ ] 什么任务 AI 能一次做对？
- [ ] 什么任务需要大量人工修正？
- [ ] 代码质量、安全性、可维护性的考量
- [ ] **产出**: 一份《AI 编程能力边界评估报告》

### 03. Vibe Coding 方法论
**目标**: 形成自己的 AI 编程工作流

- [ ] Prompt 工程：如何描述需求才能得到好代码
- [ ] 上下文管理：如何让 AI 理解项目结构
- [ ] 迭代策略：从 MVP 到完整功能的演进路径
- [ ] 代码审查：如何检查 AI 代码的质量
- [ ] **产出**: 个人《Vibe Coding 工作手册》

### 04. Dify 平台
**目标**: 掌握无代码/低代码 AI 应用开发

- [ ] Dify 安装与基础概念（应用、工作流、知识库）
- [ ] 搭建一个 Chatbot 应用
- [ ] 使用工作流编排复杂逻辑
- [ ] 接入外部 API 和数据库
- [ ] **产出**: 一个基于 Dify 的实用小工具

### 05. AI 产品原型
**目标**: 用 AI 快速验证产品想法

- [ ] 从 0 到 1 设计一个产品页面
- [ ] 用 AI 生成 UI 组件和交互逻辑
- [ ] 用户测试与反馈收集
- [ ] **产出**: 一个可演示的产品原型

### 06. 项目实践
**目标**: 综合应用所学，完成一个小项目

- [ ] 选择一个感兴趣的方向
- [ ] 用 Vibe Coding 方式完成开发
- [ ] 部署上线并收集反馈
- [ ] **产出**: 一个完整的上线项目
