# Stage-0: 基础环境 Setup

> 磨刀不误砍柴工：搭建高效的 AI 时代开发工作流

**预计耗时**: 1-2 周  
**产出目标**: 一个配置完毕、可随时进入开发状态的本地环境 + 本知识库的熟练使用

---

## 🎯 阶段目标

1. 掌握 Git 基础操作和团队协作工作流
2. 配置好主力开发环境（IDE + AI 助手）
3. 建立文档化工作习惯（Markdown + 本知识库规范）
4. 了解当前主流 AI 工具的能力边界和使用场景

---

## 📋 学习任务清单

### 1. Git 基础
- [ ] Git 安装与配置（用户名、邮箱、SSH Key）
- [ ] 核心命令：`clone`, `add`, `commit`, `push`, `pull`, `branch`
- [ ] GitHub 工作流：Fork → Clone → Branch → PR
- [ ] 常用场景：代码回滚、冲突解决、.gitignore
- [ ] **实践**: 将本知识库推送到自己的 GitHub 仓库

### 2. 开发环境配置
- [ ] 安装 Cursor / VSCode + 必备插件
- [ ] 配置 AI 助手：Claude / ChatGPT / Kimi 的 API 或客户端
- [ ] 终端工具：iTerm2 / Warp / Terminal
- [ ] Node.js / Python 环境管理（nvm / conda）
- [ ] 浏览器开发工具链

### 3. Markdown & 文档规范
- [ ] Markdown 基础语法（标题、列表、代码块、链接、图片）
- [ ] 本知识库的目录结构和命名规范
- [ ] 如何写一份好的学习笔记（见 [学习笔记模板](../../99-Templates/学习笔记模板.md)）
- [ ] **实践**: 用 Markdown 写一份个人环境配置清单

### 4. AI 工具链认知
- [ ] 主流大模型能力对比（GPT-4o / Claude 3.5 / Kimi / DeepSeek）
- [ ] Chat 模式 vs Agent 模式 vs IDE 集成模式
- [ ] Token 计费与上下文长度概念
- [ ] **实践**: 用不同模型完成同一个任务，对比输出质量
