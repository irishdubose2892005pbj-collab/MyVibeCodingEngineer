# Stage-3: Agent 智能体系统

> 从应用开发者到智能体架构师

**预计耗时**: 持续深入  
**产出目标**: 1 个具备记忆、工具调用和多 Agent 协作的复杂系统

---

## 🎯 阶段目标

1. 理解 Agent 的核心架构（感知 → 推理 → 行动 → 记忆）
2. 掌握 Workflow 编排，能设计复杂业务流程
3. 实现 Memory 系统，让 Agent 具备长期记忆
4. 理解 MCP 协议，实现工具的标准化接入
5. 探索 Multi-Agent 协作模式
6. 建立 Harness 评测体系，量化 Agent 能力

---

## 📋 学习任务清单

### Workflow
- [ ] Agent 工作流基础概念（ReAct、CoT、ToT）
- [ ] 可视化编排工具（Dify / Coze / LangGraph）
- [ ] 条件分支、循环、异常处理
- [ ] **实践**: 设计一个客服机器人的完整工作流

### Memory
- [ ] 短期记忆：对话历史管理
- [ ] 长期记忆：向量数据库（Pinecone / Milvus / Chroma）
- [ ] 记忆检索：RAG（检索增强生成）
- [ ] 记忆压缩与摘要
- [ ] **实践**: 为 Agent 添加能记住用户偏好的记忆系统

### MCP
- [ ] MCP (Model Context Protocol) 协议理解
- [ ] 工具定义与 Schema 规范
- [ ] 本地工具与远程工具的接入
- [ ] **实践**: 实现一个自定义 MCP 工具（如查询天气、操作文件）

### Multi-Agent
- [ ] 多 Agent 架构模式（分层、协作、竞争）
- [ ] Agent 通信协议
- [ ] 任务分配与调度
- [ ] **实践**: 搭建一个"产品经理 + 程序员 + 测试员"的多 Agent 团队

### Harness
- [ ] Agent 评测指标体系
- [ ] 自动化测试框架
- [ ] A/B 测试与效果追踪
- [ ] **实践**: 为自己的 Agent 建立评测数据集和自动化测试
